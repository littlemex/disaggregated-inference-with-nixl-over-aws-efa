## 6. Disaggregated Inference テスト（2026-03-07 19:00-19:15 UTC）

### 重要な発見

#### vLLM v0.17.0 で Disaggregated Inference を動作させる方法

**問題**:
- Producer/Consumer が独立して動作し、KV 転送が発生していなかった
- NIXL のログに `postWrite`/`postRead` が一切記録されていない（0 件）

**調査結果**:
1. **Proxy Server が必須**
   - vLLM v0.17.0 では Producer と Consumer を直接連携させることはできない
   - Proxy Server が Prefill と Decode を orchestrate する必要がある

2. **`kv_transfer_params` が必須**
   - Prefill リクエストに `kv_transfer_params: {"do_remote_decode": true}` を追加
   - これにより Producer が KV-cache メタデータを返す
   - Decode リクエストに Prefill の `kv_transfer_params` を渡す

3. **Proxy の実装**
   - `/home/coder/phase2/disaggregated-inference-with-nixl-over-aws-efa/experiments/scripts/disagg_proxy_server.py`
   - Phase 3 で実際に使用されている実装を発見

#### Proxy Server の動作フロー

```
[Client] → [Proxy Server] → [Producer (Prefill)] → KV-cache 生成
                          ↓
                    [Consumer (Decode)] ← NIXL で KV-cache 転送
                          ↓
           [Response] ← [Proxy Server]
```

**Phase 1: Prefill**
```json
{
  "model": "Qwen/Qwen2.5-32B-Instruct",
  "prompt": "...",
  "max_tokens": 1,
  "kv_transfer_params": {"do_remote_decode": true}
}
```

**Phase 2: Decode**
```json
{
  "model": "Qwen/Qwen2.5-32B-Instruct",
  "prompt": "...",
  "max_tokens": 20,
  "kv_transfer_params": {
    "remote_block_ids": [...],
    "agent_id": "...",
    ...
  }
}
```
