## ログ分析

### FI_RMA が依然としてリクエストされている？

Producer の LIBFABRIC ログ:
```
Requested: FI_MSG, FI_RMA, FI_LOCAL_COMM, FI_REMOTE_COMM, FI_HMEM
```

**解釈**: これは ofi_rxm (RDM エミュレーション層) が表示しているもので、実際の EFA provider は FI_MSG のみを使用している可能性がある。バイナリの MD5 が一致しているため、two-sided 版が正しくロードされている。

### postWrite/postRead が呼ばれていない

リクエスト送信後のログに `postWrite` / `postRead` / `fi_senddata` / `fi_recv` の呼び出しが一切ない。

**原因の可能性**:
1. Producer/Consumer が実際にリクエストを受信していない
2. Disaggregated inference が有効になっていない
3. NIXL 接続が確立していない
4. 設定パラメータの問題
